# 数据识别规则

## 1. 需要识别的数据类型

识别段落中所有带有统计含义的数据表达，尤其是早于2025年的数据。

常见类型包括：

1. 市场规模：市场规模、销售规模、交易规模、产值、营收、收入、增加值；
2. 供给能力：产量、产能、装机量、出货量、交付量、供应量；
3. 主体数量：企业数量、规上企业数量、上市公司数量、项目数量、园区数量、机构数量；
4. 投融资：投资额、融资额、资本支出、并购金额；
5. 增长指标：同比增长、复合增长率、增速、年均增长；
6. 结构指标：占比、份额、渗透率、集中度、国产化率；
7. 区域指标：全国、全球、省、市、区县、园区等区域数据；
8. 科技创新：专利、论文、研发投入、研发人员、高新技术企业；
9. 贸易指标：进出口额、出口额、进口额、贸易顺差；
10. 排名指标：位列第一、排名前三、全球第几、全国第几。

## 2. 需要更新的数据

默认更新以下数据：

- 明确年份早于2025年的产业数据；
- 明确年份早于2025年的市场规模、产量、产值、企业数量、投资额、增速、占比等；
- 以“近五年”“过去十年”“近年来”等表述支撑，但上下文显示数据停留在2025年前；
- “预计2025年”且当前已有2025年实际数据或更近数据。

## 3. 不需要更新的数据

以下数据默认不更新：

- 企业成立时间，例如“公司成立于2018年”；
- 政策发布时间，例如“2021年出台《××政策》”；
- 规划周期名称，例如“十四五”“2021—2025年规划”；
- 历史事实年份，例如“2019年首次实现量产”；
- 专有名词中的年份；
- 已经是2025年及之后的数据；
- 用户明确要求保留的数据。

## 4. 易错判断

- “2024年出台政策”通常不是产业统计数据，不更新。
- “2024年市场规模达到××亿元”是产业统计数据，需要更新。
- “2025年预计达到××亿元”需要优先判断是否已有实际数据；如果没有实际数据，可保留或换成更近预测。
- “截至2024年底”通常是存量数据，需要查找截至2025年底或最新公开时点的数据。
